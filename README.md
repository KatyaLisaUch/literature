# Учет сдачи стихов

Статическое веб-приложение для GitHub Pages. Данные хранятся в браузере пользователя через `localStorage`.

## Excel с учениками

Первая строка должна содержать заголовки:

- `ФИО`
- `Класс`

Пример: `Петров Петр Петрович`, `5-Т`.

## Excel со стихами

Первая строка должна содержать заголовки:

- `Название`
- `Автор`
- `Класс`
- `Дата начала сдачи`
- `Дата окончания сдачи`

В колонке `Класс` можно писать параллель: `5`, `6`, `10`. Если указать `5-Т`, приложение возьмет параллель `5`.

## Публикация на GitHub Pages

1. Создайте репозиторий на GitHub, например `poetry-tracker`.
2. Загрузите в корень репозитория файлы `index.html`, `styles.css`, `app.js`, `README.md`.
3. Откройте `Settings` -> `Pages`.
4. В блоке `Build and deployment` выберите `Deploy from a branch`.
5. В `Branch` выберите `main` и папку `/root`.
6. Нажмите `Save`.

Через 1-3 минуты страница будет доступна по адресу вида:

`https://ВАШ_ЛОГИН.github.io/poetry-tracker/`

## Ограничение первой версии

Это версия без общего сервера и без личных кабинетов. Каждый учитель работает со своими данными в своем браузере. Для общей базы Google Sheets и логинов нужно будет добавить следующий этап.
